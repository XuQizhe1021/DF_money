# 三角洲行动子弹行情分析系统 — 项目计划书

## 一、项目概述

### 1.1 项目背景

《三角洲行动》中的子弹交易市场具有明显的周期性波动特征。根据市场规律，每日6:00-9:00为价格低谷时段，周末玩家活跃高峰期价格通常会上涨[reference:0]。与此同时，赛季周期、版本更新和限时活动也会对弹药价格产生显著影响[reference:1]。

然而，普通玩家难以实时追踪所有子弹品种的价格变化趋势，更无法精准把握买卖时机。部分玩家选择使用自动化脚本进行“滚仓”操作，但这类工具违反游戏规则，存在极高的封号风险[reference:2]。

### 1.2 项目目标

开发一款**纯本地运行、不涉及账号操作**的子弹行情分析系统，通过公开API获取市场价格数据，结合AI大模型提供智能化的买卖建议，帮助用户在遵守游戏规则的前提下优化子弹交易决策。

### 1.3 核心功能

| 模块     | 功能         | 说明                                        |
| -------- | ------------ | ------------------------------------------- |
| 数据采集 | 价格定时爬取 | 通过公开API获取所有子弹品种的最低价格       |
| 数据存储 | 本地历史记录 | 使用SQLite数据库存储价格历史                |
| AI分析   | 市场行情解读 | 用户配置自己的API Key，接入DeepSeek等大模型 |
| 持仓管理 | 购买记录登记 | 记录用户购买的子弹种类、价格和时间          |
| 价格预警 | 涨价提醒     | 运行中持续监控，当前价格高于持仓成本时提醒  |
| Web展示  | 数据可视化   | 价格走势图、AI建议、持仓列表                |

### 1.4 安全性原则（红线）

> **本项目不获取任何账号信息、不登录游戏账号、不执行任何自动化交易操作。** 所有数据仅用于分析参考，买卖决策由用户自行执行。请务必遵守《三角洲行动》用户协议，避免使用任何自动化交易脚本。

### 2.3 核心模块详解

#### 2.3.1 数据采集模块（DataFetcher）

**职责**：从三角洲行动公开API获取子弹市场价格数据。

**实现要点**：
- 调用玩家社区整理的公开API（如 df-api.apifox.cn），获取所有子弹品种的当前最低价[reference:4]
- 通过社区贡献的API接口文档进行数据查询，该文档由社区开发者整理并提供后端支持[reference:5]
- 定时任务每日多次抓取，存入本地数据库
- 抓取字段：子弹ID、子弹名称、当前最低价、更新时间

**技术细节**：
```python
# 伪代码示例
class DataFetcher:
    def fetch_all_ammo_prices(self):
        # 调用API获取数据
        # 解析JSON响应
        # 存入price_history表
        pass
```

#### 2.3.2 数据存储模块（Database）

**职责**：使用SQLite持久化存储价格历史和用户持仓，同时支持用户进行删除，前端做好适配。

**表结构设计**：

**表1：ammo_info（子弹信息表）**
| 字段    | 类型       | 说明                         |
| ------- | ---------- | ---------------------------- |
| id      | INTEGER PK | 子弹唯一标识                 |
| name    | TEXT       | 子弹名称（如5.56x45mm）      |
| grade   | TEXT       | 等级（紫色/金色等）          |
| caliber | TEXT       | 口径类型（此字段可考虑忽略） |

**表2：price_history（价格历史表）**
| 字段        | 类型       | 说明                 |
| ----------- | ---------- | -------------------- |
| id          | INTEGER PK | 自增主键             |
| ammo_id     | INTEGER FK | 关联子弹ID           |
| price       | INTEGER    | 当前最低价（哈夫币） |
| recorded_at | DATETIME   | 记录时间戳           |

**表3：holdings（持仓记录表）**
| 字段           | 类型       | 说明               |
| -------------- | ---------- | ------------------ |
| id             | INTEGER PK | 自增主键           |
| ammo_id        | INTEGER FK | 关联子弹ID         |
| purchase_price | INTEGER    | 购买时的价格       |
| purchased_at   | DATETIME   | 购买时间           |
| status         | TEXT       | 状态：holding/sold |

> **说明**：持仓表只记录**子弹种类**和**购买价格**，不记录具体数量，符合用户需求。

#### 2.3.3 AI分析模块（AIAnalyzer）

**职责**：读取历史价格数据，调用大模型API生成买卖建议。

**实现要点**：
- 用户自行配置API Key（支持DeepSeek、OpenAI、Claude等）
- DeepSeek API调用示例：通过Python发送HTTP请求至 `https://api.deepseek.com/chat/completions`[reference:6]
- 用户首次注册DeepSeek可获得一定的免费额度，后续按Token用量计费，输入缓存命中仅需0.2元/百万tokens[reference:7]
- 系统向AI发送价格数据和分析提示词，AI返回结构化建议

**AI提示词设计**：
```
你是一个《三角洲行动》市场分析师。以下是某子弹的历史价格数据：
[数据展示]
请根据市场规律（每日6-9点为价格低谷，周末为价格高峰）给出：
1. 当前价格所处位置（高位/低位/持平）
2. 操作建议（买入/卖出/观望）
3. 风险提示
```

**支持的AI模型**：
- DeepSeek（推荐，性价比高）
- OpenAI GPT系列
- Claude系列
- 其他兼容OpenAI API格式的模型

#### 2.3.4 价格监控与提醒模块（PriceMonitor）

**职责**：在项目运行时持续监控价格，对比用户持仓成本，当价格高于购买价时发出提醒。

**实现要点**：
- 后台守护线程/定时任务持续运行
- 每次获取最新价格后，遍历用户持仓记录
- 若当前价格 > 持仓成本价 ×（1 + 用户设定的盈利阈值），触发提醒
- 提醒方式：Web页面弹窗通知 / 控制台输出 / 系统通知

**提醒逻辑示例**：
```
持仓：5.56x45mm，购买价120哈夫币
当前价：156哈夫币 → 涨幅30% → 触发提醒
建议：可考虑卖出
```

#### 2.3.5 前端展示模块（WebUI）

**职责**：提供直观的Web界面，展示价格数据和分析结果。

**功能页面**：
1. **行情看板**：当前所有子弹价格排行、涨跌幅排行
2. **价格走势图**：7日/30日价格曲线
3. **AI建议面板**：展示AI生成的买卖建议
4. **持仓管理**：增删改查持仓记录，查看每种子弹的盈亏状态

**前端实现建议**：使用Vue.js 3搭建前端界面，后端通过RESTful API提供数据接口[reference:8]。


## 三、功能模块详解

### 3.1 子弹价格数据获取与统计

- **定时频率**：默认每小时抓取一次（用户可配置）
- **数据范围**：涵盖游戏内所有可交易子弹品种
- **统计维度**：
  - 当前最低价
  - 7日均价、30日均价
  - 日涨幅、周涨幅
  - 历史最高价/最低价
- **数据源**：使用社区整理的三角洲行动公开API[reference:9]

### 3.2 AI辅助分析市场行情

- **触发方式**：用户手动点击“分析”按钮，或系统定时自动分析
- **分析输入**：选定子弹品种的历史价格数据（最近7-30天）
- **分析输出**：
  - 当前价格评估（偏高/偏低/合理）
  - 短期走势预测（结合市场规律，如每日6-9点低价期、周末高价期[reference:10]）
  - 具体操作建议（买入/卖出/持有/观望）
  - 风险等级提示

### 3.3 购买与售卖建议

- **AI建议**：基于历史数据和分析规律生成
- **技术指标辅助**（可选扩展）：
  - 移动平均线（MA）趋势判断
  - 相对强弱指数（RSI）超买超卖判断
  - 价格突破提醒
- **赛季周期考量**：赛季结束前1个月通常为高价期，赛季初与赛季末为低价期[reference:11]

### 3.4 用户持仓记录

- **功能入口**：Web界面“我的持仓”页面
- **记录内容**：子弹种类、购买价格、购买时间
- **操作功能**：添加持仓、编辑成本价、删除（卖出）记录
- **展示信息**：当前价、盈亏比例、建议操作
- **数据存储**：仅保存在本地SQLite数据库，不上传任何云端

### 3.5 涨价提醒功能

- **运行方式**：仅项目运行时生效，不持久化后台运行
- **监控频率**：每次API数据更新后自动对比
- **提醒条件**：当前价格 > 持仓成本价（用户可自定义盈利触发百分比）
- **提醒方式**：
  - Web页面顶部通知条
  - 浏览器桌面通知（需用户授权）
  - 控制台彩色输出
- **与AI建议联动**：触发提醒时可同时展示AI对该子弹的最新分析

## 六、关键技术难点与解决方案

### 6.1 公开API的稳定性与认证

**难点**：社区整理的API可能需要openid和access_token才能访问[reference:12]。

**解决方案**：
- 用户需自行通过三角洲官网扫码登录获取凭证[reference:13]
- 项目提供详细的图文教程，引导用户完成凭证获取
- 将凭证存储在本地配置文件中，不上传任何云端

### 6.2 AI分析的成本控制

**难点**：频繁调用AI API会产生费用。

**解决方案**：
- 采用DeepSeek API，性价比极高（输入缓存命中0.2元/百万tokens）[reference:14]
- 用户可自行选择是否启用AI分析，也可以仅使用技术指标判断
- 设置分析频率限制（如每小时最多5次）
- 缓存历史分析结果，避免重复请求

### 6.3 价格数据的实时性与准确性

**难点**：游戏交易行每5秒刷新一次价格数据，系统无法达到同频率[reference:15]。

**解决方案**：
- 明确告知用户数据存在一定延迟（1小时级），仅供参考
- 提供手动刷新按钮，满足即时查询需求
- 标注数据更新时间，让用户知晓数据新鲜度

### 6.4 用户数据的安全性

**难点**：持仓记录涉及用户个人决策信息。

**解决方案**：
- 所有数据纯本地存储，不上传任何服务器
- 项目开源，用户可审计代码
- 提供数据导出和清除功能

## 十、总结

本项目旨在为《三角洲行动》玩家打造一款**安全、免费、开源的子弹行情分析工具**。通过社区公开API获取价格数据，结合AI大模型提供智能建议，帮助用户更理性地决策。

**项目核心价值**：
1. **安全性** — 不登录、不操作，无封号风险
2. **智能化** — AI辅助分析，降低决策门槛
3. **本地化** — 数据存储本地，保护用户隐私
4. **可扩展** — 开源设计，社区可共同维护

按照上述计划，预计**8周内**完成第一个可交付版本。如需进一步细化某个模块的技术实现方案，或调整计划书的优先级安排，请随时告知。
